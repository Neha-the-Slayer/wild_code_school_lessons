A list of essential git commands

  • git init: Initialized empty Git repository
  • git add .: Add to staging what will be committed excluding deletions
  • git diff : difference between working directory and staging. Use flag --staged(or cached) to get differences in the staging area
  • git reset HEAD < filename > : remove from staging. Just "git reset" to remove all form staging
  • git commit -m: Commit a version to git. The "-m" allows you to specify a message to add info about the commit
  • git checkout < filename >: Discard changes in working directory
  • git checkout < branch >: Switch to a different branch
  • git checkout -b < new branch name > < Log Code >: When checked out in a previews Log, create a new branch
  • git checkout -b < new branch name > origin < Existing Branch Name >: When Pulling somone elses repo, it will only pull master. Issue the command to pull a specific branch name in a new branch
  • git merge < branch >: merge < branch > in the branch you are currently in
  • git branch: Show all branches available
  • git branch < name >: Create a new branch. This is like saving the state of everything at a given point
  • git branch -m < original name> < new name >: to change the name of a branch
  • git branch -D < branch name >: To delete a branch
  • git rm < /filename >: delete a file and automatically moves it to the staging environment ready to be committed
  • git rm --cached < /filename >: To stop tracking a file that is currently tracked
  • git mv < /filename >: rename a file and automatically moves it to the staging environment ready to be committed
  • git clone < repository URL >: Copy a git repository on you local machine. Will only pull the master branch
  • git clone -b < branch name > < repository URL >: Copy a specific branch from a git repository on you local machine
  • git log: A list of all commit made for the project
  • git push: After committing, push the changes to the remote repo
  • git push --set-upstream origin < branch Name >: After creating a branch and after committing, push the changes and the new branch to the remote repo

Clone a full repository with all branches

  1. git clone --mirror < repository URL >.git: Copy a git repository on you local machine including all Branches.
  2. The command above will copy the invisible .git folder that keeps track of everything that happens in a repository.
  3. Next run, git config --bool core.bare false: This will modify a boolean value that will take the repo from being a bare repository into a regular one
  4. Next run, git reset --Hard: This is a hard reset that will take everything that is inside the .git and create all the branches makeing it look like normal repository

Git Log Flags

  1. git log -n < number >: List of the last n commits
  2. git log --since=< yyy-mm-dd >: Will return the commit since a certain date.
  3. git log --until=< yyy-mm-dd >: Will return the commit untill a certain date. It can be used together with since
  4. git log --author=< any part of a name >: Will return the commits from authors whose name matched the string
  5. git log --author=< any part of a name >: Will return the commits from authors whose name matched the string

Git Concpts and architecture

Hash Values (SHA-1)

git keeps track of change sets. For each of them it generates a checksum (or hash values). This is generated by taking data and feeding it to a mathematical alogrithm (SHA-1) which converts the data into a number. This number is called checksum: The same data, put in the same algorithm, always return the same result (checksum). This guarantee data integrity. The checksum is a 40 character hexadecimal string(0-9,a-f).

The HEAD pointer

The HEAD pointer is a reference variable that points to a specific commit in the repository: The parent of the next commit that we'll make (or The Tip of the current branch in the repository).

git push -u origin

The -u option is important because it binds your local branch to the remote branch. origin is the name of the remote which we gave to it when we ran git remote add. master is the name of the branch created on the remote, the same as your current local branch. Note: the -u option is to be used only during your first push of this branch. After entering the command, your local and remote master branches are linked. With the future pushes from your local master branch, Git will know that it must push to the remote master branch.

It is possible to use the push in several ways depending on the arguments, but at the moment, just remember that we specify the remote and the branch the first time with the -u flag, and that thereafter, git push is enough
